# Project: Page of Playing Cards

This project is a certification task within the **freeCodeCamp Responsive Web Design** curriculum. The focus is on creating a structured, multi-element layout using modern CSS techniques to simulate a realistic deck of playing cards.

## 🛠 Design & Implementation
- **Layout Architecture:** Used CSS Flexbox for the overall grid and individual card alignment.
- **Responsive Geometry:** Implemented `flex-wrap` and `max-width` constraints to achieve a balanced 3-2 distribution across the viewport.
- **Visual Depth:** Applied `radial-gradient` for the background and `box-shadow` for the cards to create a layered, professional UI.
- **Component Styling:** Used `transform: rotate()` to handle the inverted orientation of the bottom suit symbols.

## 📋 Technical Requirements
- [x] Structured each card with `.left`, `.middle`, and `.right` divisions for semantic clarity.
- [x] Applied color-specific CSS classes (`.red`, `.black`) to differentiate card suits.
- [x] Managed precise alignment using `align-self` and `justify-content` properties.
- [x] Verified cross-device responsiveness for a consistent user experience.

## 💡 Project Insights
The main challenge of this project was mastering the **Flexbox coordinate system**. Aligning different elements within the same container—some at the top, some at the bottom, and one perfectly centered—required a solid understanding of how flex properties interact. Adhering to the specific naming conventions also reinforced the importance of following a strict design specification.
