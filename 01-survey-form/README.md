# Project: Survey Form - Art & Design Feedback

This project is a certification requirement for the **freeCodeCamp Responsive Web Design** curriculum. The objective was to build a fully functional, accessible, and responsive survey form to collect user feedback.

## 🛠 Design & Implementation
- **Form Architecture:** Utilized semantic HTML5 form elements, including various input types (text, email, number), dropdown menus, radio buttons, and checkboxes.
- **Styling & UI:** - Implemented a clean, modern aesthetic with a focus on typography and spacing.
  - Used a soft color palette to maintain a creative and professional feel.
  - Applied `box-shadow` and `border-radius` to the form container for a card-based layout.
- **Responsiveness:** Managed flexible widths and `box-sizing` to ensure the form is usable across different screen sizes.
- **UX Features:** Included `:hover` states for buttons and clear label-input associations for better accessibility.

## 📋 Technical Requirements
- [x] Defined a survey form with a unique `id="survey-form"`.
- [x] Included required `input` fields for Name and Email with appropriate validation.
- [x] Implemented a `number` input with defined `min` and `max` attributes.
- [x] Integrated a `select` dropdown and groups of `radio` and `checkbox` inputs.
- [x] Used `textarea` for open-ended user comments.
- [x] Styled a dedicated submit button with transition effects.

## 💡 Project Insights
The main focus of this project was to master **HTML Form Validation** and **Layout Consistency**. Ensuring that all input types were correctly labeled and styled required careful attention to CSS selectors. This task reinforced the importance of creating a user-friendly interface that guides the participant through the data entry process smoothly.
